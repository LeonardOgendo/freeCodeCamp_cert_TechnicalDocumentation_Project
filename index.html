Introduction

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design.

It was designed to help developers take applications from concept to completion as quickly as possible.

Django takes security seriously and helps developers avoid many common security mistakes.

The framework is free and open-source, with a large community of developers building and maintaining it.

Here are some key features of Django:

  • Object-Relational Mapping (ORM)
  • Automatic admin interface
  • Elegant URL design
  • Template system
  • Form handling
import django
Installation

To install Django, you need to have Python installed on your system.

Using pip, you can install Django with the following command:

pip install django

Once installed, you can verify the installation by checking the version:

django-admin --version

Django also supports virtual environments to keep dependencies isolated.

Models

Models are the single, definitive source of information about your data.

They contain the essential fields and behaviors of the data you’re storing.

Each model maps to a single database table.

Here's an example of a simple model:

from django.db import models class MyModel(models.Model): name = models.CharField(max_length=100)

After defining models, you can create database tables by running:

python manage.py migrate
Views

Views are Python functions that take a web request and return a web response.

They can read records from a database, generate HTML, and return it as a response.

Here’s an example view:

from django.http import HttpResponse def my_view(request): return HttpResponse('Hello, World!')

To link views to URLs, you need to define URL patterns.

from django.urls import path urlpatterns = [ path('my-view/', my_view), ]
Templates

Templates define the structure of the HTML documents that a view returns.

They can include dynamic content using Django’s template language.

Here is a simple template example:

<h1>{{ title }}</h1> <p>{{ content }}</p>

Templates are usually stored in a directory named 'templates' within your app.

You can render a template in a view like this:

from django.shortcuts import render def my_view(request): return render(request, 'my_template.html', {'title': 'My Title', 'content': 'My Content'})
